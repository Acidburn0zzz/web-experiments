Selection Mode

Selection Mode is a pattern to allow for the selection of multiple items of content and to allow batch actions to performed on that selection. It would typically be used in conjunction with long lists or grids of items. Appropriate scenarios for Selection Mode would be when the view has multple content items and:

  • You need to perform actions simultaneously.
  • Several actions are available to be performed.
  • It is helpful to be able to perform actions without opening them.
  • You need to organize or reorder items.

Activating Selection Mode

Selection Mode is activated by a right-aligned button, identified either with a text label Edit... or a icon {%- include symbolic/emblems/emblem-ok-symbolic.svg -%}, which transforms the header bar into the Selection Mode (by hiding the main window actions and revealing the Selection Mode actions).

Selection Mode would be de-activated when the batch action is performed or when cancelled or dismissed, typically with a button: Cancel or Done.



Selection Mode Actions

While in Selection Mode, the title bar contains actions that only apply to a batch selection of items.

Actions specific to the application occupy left-aligned buttons with text labels—it's crucial to avoid the use of icons for these app-specific actions are they are often only known to the app. Standard, more universal actions, that may be used across applications, occupy right-aligned buttons with icons—icons are used here because the actions often are widely used, e.g. Print ({%- include symbolic/actions/document-print-symbolic.svg -%}).
